이슈: 서버 런타임 라이브러리 미인식으로 인한 PageContext 참조 오류 발생.

해결: IDE 내 Targeted Runtimes 설정을 통해 서버 API 라이브러리(Standard Tag Library) 의존성 문제 해결.


1. 사용자가 목록에서 대여 요청(item_id 전달). 
2. 컨트롤러가 요청 수신 후 서비스 계층으로 전달. 
3. 트랜잭션 관리: 재고 수량 확인 및 차감(UPDATE)과 대여 이력 생성(INSERT)을 하나의 작업 단위로 묶어 데이터 결함 방지.

구현 기능: 비품 대여 로직 (Inventory Rent Process)
기술적 포인트:
Transaction 관리: @Transactional을 사용하여 수량 차감(UPDATE)과 대여 이력 삽입(INSERT)의 원자성(Atomicity) 확보.
MVC 패턴 완결: JSP → Controller → Service → Mapper → DB로 이어지는 데이터 흐름 완성.
예외 처리: DB 수준에서 quantity > 0 조건을 걸어 마이너스 재고 발생 방지.

PPT [4. 핵심 기능 구현] 혹은 [UX/UI 설계] 페이지에 이렇게 적으시면 좋습니다.
항목: 사용자 내비게이션 최적화
내용: "대여 현황 확인 후 전체 목록으로 즉시 복귀할 수 있도록 직관적인 '돌아가기' 버튼 배치 및 절대 경로 지정을 통한 링크 안정성 확보."

PPT 기록용 한 줄
구현 핵심: "데이터 정합성 유지를 위해 @Transactional을 적용하여 대여 반납일 업데이트와 재고 수량 복구 작업을 원자적으로 처리함."

구분,발생했던 문제,해결 방법
경로 중복,/inventory/inventory/list 발생,@RequestMapping 구조 이해 및 상대 경로(redirect:list) 도입
무한 루프,Too Many Redirects 에러,redirect와 일반 view 리턴의 차이점을 파악하여 로직 수정
데이터 모델,데이터 전달 방식 고민,Map 대신 구조화된 RentalVO를 직접 설계하여 데이터 안정성 확보
트랜잭션,수량 변경과 기록 추가의 불일치 위험,@Transactional을 사용하여 DB 작업의 원자성(All or Nothing) 보장
