📋 [비품 관리 프로젝트] 기술 스냅샷 (Technical Snapshot)
1. 기술 스택 (Tech Stack)

Framework: Spring MVC + MyBatis

Language: Java

Database: MySQL

View: JSP (JSTL: c:forEach, c:if, fmt 사용)

2. DB 구조 (Table Schema)

item: 비품 정보 (item_id, item_name, cat_id, quantity, status, reg_date)

status: 'active' (정상), 'deleted' (논리 삭제됨)

item_category: 카테고리 정보 (cat_id, cat_name)

rental: 대여 기록 (rental_id, emp_id, item_id, rent_date, ret_exp_date, ret_act_date)

emp: 사원 정보 (emp_id, name)

3. 핵심 비즈니스 로직 (Core Logic)

논리 삭제 (Soft Delete): 데이터를 실제로 지우지 않고 status = 'deleted'로 변경하여 관리자 페이지와 대여 이력에 남김.

재고 연동: 대여 시 재고 감소, 반납 시 재고 증가. 단, 삭제된 비품(status='deleted')을 반납할 때는 재고를 증가시키지 않음.

데이터 매핑: RentalVO 내에 item_name, emp_name, status 필드를 두어 JOIN 결과 데이터를 수용함.

4. 오늘 추가된 기능 (Updates)

동적 카테고리 연동: itemRegForm에서 DB의 item_category 데이터를 불러와 <select> 드롭다운으로 자동 생성.

관리자 전체 대여 관리: rental, item, emp 3개 테이블을 JOIN하여 모든 대여 이력을 조회하는 페이지 구현 완료.

UI 개선: 대여 목록 하단에 단순 '돌아가기' 버튼 추가.

5. 다음에 할 일 (To-Do)

아이템 수정 페이지(editForm)에도 동적 카테고리 드롭다운 적용 및 기존 값 유지 로직 추가.

대여 목록에서 연체된 항목(오늘 날짜 기준) 시각적 강조 처리.
