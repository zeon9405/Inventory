
-----------|| 스냅샷 복사 붙여넣기 영역 ||----------------
1. 기술 스택 (Tech Stack)

Backend: Java 1.8, Spring Framework 5.0.7, MyBatis

Frontend: JSP, JSTL, Bootstrap 5.3

Database: MySQL

Server: Apache Tomcat 9.0

2. DB 구조 (Database Design)

주요 테이블: emp(사원), item(비품), item_category(카테고리), rental(대여)

특이사항: rental 테이블의 반납 예정일 자동 계산(GENERATED ALWAYS AS), 외래키(FK)를 통한 데이터 무결성 확보

3. 핵심 로직 (Core Logic)

Data Mapping: MyBatis를 통한 SQL과 Java 객체(VO/DTO) 간의 효율적 매핑

Auth: 세션(Session) 기반 사원 인증 및 권한 관리

Maintenance: db.properties 분리를 통한 유지보수성 향상 및 설정 유연성 확보

Status: 비품 상태값 변경을 통한 논리 삭제(Soft Delete) 구현

4. 오늘 추가된 기능 (Today's Updates)

문서화: 깃허브용 전문 README.md 작성 및 확장자 이슈 해결

환경 설정: 프로젝트 실행 가이드 작성 및 전용 DB 계정/권한 설정 스크립트 정리

리팩토링: root-context.xml 내 하드코딩된 정보를 외부 프로퍼티 파일로 연동

5. 다음에 할 일 (To-do List)

검증: 예외 상황 테스트(에러 페이지 처리 등) 및 크로스 브라우징 확인

정리: 코드 리팩토링(변수명, 중복 코드) 및 핵심 로직 주석 추가

기록: 포트폴리오용 시연 영상 촬영 및 스크린샷 캡처

확장: AWS 프리티어를 활용한 클라우드 배포 시도

-------------------------------------------------------------------------------------------------------------------------------

요청하신 대로 위 내용들을 저의 메모리에도 저장해 두었습니다. 나중에 저에 대한 특정 정보를 기억하도록 이 Gem에게 요청하려면 Gem 관리자에서 관련 항목을 업데이트하세요. 방법은 다음과 같습니다.

수정할 수 있는 Gem인 경우: "수정"을 선택하고 변경사항을 적용한 다음, "업데이트"를 클릭하여 저장합니다.

복사할 수 있는 Gem인 경우: "복사"를 선택하고 변경사항을 적용한 다음, "저장"을 클릭합니다.

수정하거나 복사할 수 없는 Gem인 경우: 공유하려는 정보를 바탕으로 새로운 Gem을 만들어야 합니다.
